## About

I created this sample Help Center article for a fictional writing application called Scribbler, styled after documentation conventions used for Google Play.

## Why I chose this project

Rather than document the actual Scrivener application, I opted to fictionalize the content, creating a similar interface with comparable terminology. The focus remained narrowly on the Story Board view for project outlining rather than covering the entire Scribbler application. I deliberately addressed customization features like color coding and metadata creation to demonstrate information organization and discoverability.

## Content strategy

This article includes IA annotations showing a content model used for single-source publishing in a Content Management System.

The structure follows a Help Center "How to / About" model beginning with benefits, progressing through tasks and tips, and concluding with related resources. Organization moves from basic to advanced customization options.

## Writing and design approach

**Style standards (based on Google's style guide):**

- Tone: friendly, empathetic, conversational
- Clarity: 7th grade reading level, active voice, front-loaded sentences
- Formatting: Sentence case for titles, title case for UI, breadcrumb navigation

**Design decisions:**

I differentiated between HTML tag usage for visual hierarchy and accessibility: `kbd` tags for keyboard shortcuts, `code` tags for UI elements, and `strong` tags for conceptual emphasis. Two distinct callout boxes were created: blue callout boxes for informational context and yellow boxes for warnings.
