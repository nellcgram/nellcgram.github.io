## About

This sample Help Center article documents a fictional writing application called Scribbler, following documentation patterns used by Google Play.

## Why I chose this project

Rather than document an existing application like Scrivener, the author created a fictionalized interface with similar terminology. The scope focuses specifically on the Story Board view for project outlining rather than comprehensive application coverage. The article emphasizes customization features such as color coding and metadata creation to showcase information organization and discoverability.

## Content strategy

The article includes [information architecture annotations](pdfs/hc_scribbler.pdf) demonstrating a content model designed for single-source publishing within a Content Management System.

The organizational structure follows a Help Center "How to / About" framework that begins with benefits, progresses through tasks and tips, and ends with related resources. Content moves from fundamental to advanced customization options.

## Writing and design approach

**Style standards (based on Google's style guide):**

- Tone: friendly, empathetic, conversational
- Clarity: 7th grade reading level, active voice, front-loaded sentences
- Formatting: Sentence case for titles, title case for UI, breadcrumb navigation

**Design decisions:**

The author differentiated HTML tag usage for both visual hierarchy and accessibility: `kbd` tags for keyboard shortcuts, `code` tags for UI elements, and `strong` tags for conceptual emphasis. Two callout box styles were implemented — blue for informational context and yellow for warnings.
