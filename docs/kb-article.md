## Project overview

I created this sample internal Knowledge Base article for library circulation staff, drawing from publicly-available Los Angeles Public Library policies. It demonstrates agent-facing documentation designed to support customer service teams.

## Content strategy

**Organization:** The article prioritizes agent workflows, sequencing from routine tasks (renewals) through uncommon scenarios (lost cards, damaged items) for efficient navigation.

**Scripted responses:** Templated answers and canned responses provide starting frameworks for customer interactions, similar to support resources developed for Google teams.

## Writing and design approach

**Visual hierarchy for reference:**

- HTML code tags for interface elements
- Expandable sections to minimize cognitive burden
- Color-coded reference boxes highlighting critical information

**Style standards (based on Google's style guide):**

- Tone: conversational, empathetic, welcoming
- Clarity: accessible reading level, active voice, prioritized information
- Formatting: sentence case headers, proper capitalization for UI elements, navigation breadcrumbs
- Responses emphasize customer empathy

## Key features

- Task-based workflows for typical circulation scenarios
- Eligibility reference table for rapid policy verification
- Template email and chat responses with customizable placeholders
- Metrics reporting guidance for escalation decisions
